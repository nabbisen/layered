# Implementation Handoff: RFC-057 — Multi-Document Workspace and Tabs (v2)

**Project:** omriss — Omriss Editor
**RFC:** RFC-057 — Multi-Document Workspace and Tabs (v2)
**Milestone:** M13 — Multi-Document Workspace
**Document type:** Developer implementation handoff
**Related RFC:** RFC-057 (`../../proposed/057-multi-document-workspace-and-tabs.md`)
**Lifecycle status:** inherited from RFC-057
**Primary audience:** Rust developers, Dioxus app developer, UI/UX implementer, QA engineer
**Depends on (build):** RFC-001, RFC-014, RFC-015, RFC-016, RFC-036
**Integrates with (not build prerequisites):** RFC-050, RFC-053

---

## 1. Purpose

This handoff turns RFC-057 **v2** into an implementation plan for adding a
multi-document workspace to omriss. It supersedes the v1 handoff and folds in
the architect and dev-team review decisions.

The feature introduces a workspace layer with one tab per open document. Each
tab owns one `EditorSession` and all document-specific transient state. The
visible UI exposes a tab strip only when two or more documents are open,
preserving the current single-document experience.

## 2. The twelve constraints (treat as non-negotiable)

These come from RFC-057 v2 § 0.1 and are implementation constraints, not
suggestions:

1. Invalid structured drafts **block** tab switching (no commit, no revert).
2. RFC-057 owns a **minimal draft-validity seam**; RFC-050/RFC-053 are
   **integration points, not build prerequisites**. Tabs ship for Markdown
   before JSON/TOML editing exists.
3. **Save All validates only the active draft**, then saves dirty tabs.
4. Hot draft text must **not** write one large `Signal<Workspace>` per
   keystroke.
5. Stable **`TabId`**; modal/async actions target `TabId`, not index.
6. Tab identity uses **`DocumentIdentity`**, not `FileIdentity`; only saved
   paths dedupe; untitled documents never dedupe.
7. **`preview_open` is per-tab.**
8. **Search** state closes/clears on tab switch; stale results never shown.
9. **Status** is active-tab scoped; stale messages clear on switch.
10. **Shortcuts** verified collision-free against `keyboard::interpret`
    (RFC-014) and the palette (RFC-022) before landing.
11. **clippy** is recommended, not a new hard gate, unless RFC-040/CI says so.
12. Friendlier labels: **"Close without saving" / "Quit without saving"**.

## 3. Build-vs-integration dependency (read first)

RFC-057 must be buildable for the Markdown-first product **without** JSON/TOML
adapters. Concretely:

- RFC-057 defines the `FocusedDraft` trait and `DraftApplicability` enum
  (RFC-057 v2 § 6.5). This is the seam.
- The Markdown body editor implements `FocusedDraft`; its `applicability()`
  returns `Applicable` (or `Clean`). It never returns `Invalid`.
- Tab switch / save / preview / close call `applicability()` and act on the
  result. The `Invalid` branch exists and is unit-tested **with a fake
  validator**, but is not user-exercised until RFC-054/RFC-055.

Do **not** add a hard dependency on RFC-050/RFC-053 anywhere in the build. If
you find yourself needing an adapter to compile RFC-057, stop — the seam is in
the wrong place.

## 4. Ownership map

```text
omriss        document/session operations; NO workspace/tab/TabId concept
omriss-ui     Workspace, Tab, TabId, DocumentIdentity, FocusedDraft seam,
              workspace mutators + unit tests
omriss-app    Dioxus tab strip, modal wiring, shortcuts, shell integration,
              local hot-draft signal
```

If `EditorSession` lives in `omriss-ui` today (it does), keep it there and add
`Workspace` alongside it.

## 5. Data model

### 5.1 TabId

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, PartialOrd, Ord)]
pub struct TabId(u64);
```

- Generated by `Workspace` from a monotonic counter; never reused in a process.
- Not persisted in v1. Never shown in normal UI.
- Targets tab-close, save, Save All, and modal actions.

### 5.2 Tab

```rust
pub struct Tab {
    pub id: TabId,
    pub session: EditorSession,
    pub saved_mtime: Option<SystemTime>,
    pub selected_card: usize,
    pub preview_open: bool,        // per-tab (constraint #7)
    pub search_state: TabSearchState,
}
```

Note: there is **no** `draft: Signal<String>` inside `Tab`. The hot draft is a
focused local signal in the editor panel (constraint #4, § 7). The tab's
document already holds committed body text in its `EditorSession`.

### 5.3 Workspace

```rust
pub struct Workspace {
    tabs: Vec<Tab>,
    active: Option<TabId>,   // None iff tabs.is_empty()
    next_tab_id: u64,
}
```

Public API is id-first (see RFC-057 v2 § 6.3). `activate_index` exists only for
`Ctrl+1..9`; everything else takes `TabId`. Unknown-id operations return
`WorkspaceError::NoSuchTab`, never panic.

### 5.4 DocumentIdentity (constraint #6)

```rust
pub enum DocumentIdentity {
    SavedPath(NormalizedPath),
    Untitled(TabId),
}
```

- Only `SavedPath` participates in dedupe.
- `NormalizedPath`: canonicalize where possible; resolve symlinks safely;
  platform case-fold on Windows/macOS; normalize Windows path prefixes; on
  canonicalization failure fall back to the absolute normalized path (do not
  collapse distinct files).
- Untitled buffers never dedupe. Display path is stored separately for the tab
  label and is never used for dedupe.

### 5.5 Draft-validity seam (constraint #1, #2)

```rust
pub enum DraftApplicability {
    Applicable,
    Invalid { message_key: &'static str },
    Clean,
}

pub trait FocusedDraft {
    fn applicability(&self) -> DraftApplicability;
    fn apply(&mut self) -> Result<(), EditError>; // call only when Applicable
}
```

Markdown body editor: `applicability()` is `Applicable`/`Clean`, never
`Invalid`. Structured editors (future) return `Invalid` with a `message_key`.

## 6. State ownership

### 6.1 Per-tab (inside `Tab`)

`EditorSession` (with view/focus/undo/redo), `saved_mtime`, `selected_card`,
`preview_open`, `search_state`.

### 6.2 App-global (singular)

`locale`, `settings`, `recent_files`, `palette_open`, `modal` host,
`search_open` visibility, `status` host (active-tab scoped).

### 6.3 Search (constraint #8)

v1 rule: on tab switch, **close the search panel and clear results**. Per-tab
search persistence is a deferred follow-up.

### 6.4 Status (constraint #9)

Single host, active-tab scoped. On tab switch, clear transient success/error
messages from the previous tab (or replace with the incoming tab's neutral
status). Never let "Could not save" from tab A linger on tab B.

## 7. Reactive implementation (constraint #4)

```text
Workspace signal (metadata only): tab order, active TabId, titles, dirty
  markers, paths, preview flags. Written on: open / new / close / activate /
  save result / external-mtime change / dirty transition.

Focused editor LOCAL signal: hot draft text while typing. Written on every
  keystroke. NOT part of the workspace signal.

Commit boundaries (blur / save / preview / tab switch / close): ask the active
  editor applicability(); if Applicable, apply() into the active tab's session.
```

Required outcome: typing in the active editor does not re-render the tab strip
or inactive tabs. This is the same local-draft pattern the single-document app
uses today; keep it.

## 8. Save, close, and quit lifecycle

### 8.1 Close clean tab

```text
remove tab -> next active = right neighbor, else left -> if none left, Welcome
```

### 8.2 Close dirty tab (constraint #12)

```text
Save changes before closing?
“spec.md” has changes that are not saved.
[Save] [Close without saving] [Cancel]
```

- Cancel keeps the tab open.
- Save: apply valid draft, save, then close on success.
- Close without saving: close after confirmation.
- Invalid draft: Save blocked; focus returns to the field.
- Modal carries the target `TabId`.

### 8.3 Open / New

Append tabs. No old single-document unsaved guard. Open dedupes by identity.

### 8.4 Switch (constraint #1)

```text
applicability():
  Invalid -> stay on current tab; focus field; show guidance; do NOT switch
  Applicable -> apply(); switch
  Clean -> switch
```

### 8.5 Quit (constraint #12)

```text
Save changes before quitting?
3 documents have changes that are not saved.
[Save All] [Quit without saving] [Cancel]
```

### 8.6 Save All (constraint #3)

Because switching blocks invalid drafts, inactive tabs cannot hold an invalid
uncommitted draft. So Save All validates only the **active** draft:

```text
1. Apply/validate the ACTIVE tab's pending draft.
   - Invalid -> block Save All; focus the field;
     "Fix this value before saving all documents."
2. Save each dirty tab with a saved path, in tab order.
3. For each dirty Untitled tab, run Save As sequentially.
   - user cancels a Save As -> stop; quit cancelled; saved tabs stay saved.
4. External-modification conflict -> stop; activate that tab; show the
   existing conflict dialog scoped to it.
5. Save failure -> stop; activate the failed tab; friendly message; app stays
   open; previously-saved tabs remain saved.
```

Do **not** scan inactive tabs for invalid drafts — by construction there are
none. Do **not** silently quit after a partial failure.

## 9. Keyboard shortcuts (constraint #10)

```text
Ctrl+Tab / Ctrl+Shift+Tab     Next / previous tab
Ctrl+1 .. Ctrl+9              Activate tab by position (9 = last)
Ctrl+W / Cmd+W               Close current tab (through the guard)
Ctrl+N / Cmd+N               New document tab
```

Existing O/S/Shift+S keep their bindings and act on the active tab.

**Verified at handoff authoring:** the current `keyboard::interpret` map binds
O, S, Shift+S, Z, Y, Shift+Z, Alt+arrows, Escape, Enter, Up, Down, Backquote,
F, P, Shift+P. `KeyW`, `Tab`, the digit codes, and `KeyN` are unmapped, so the
tab shortcuts are collision-free. **Re-verify against the code** in the
shortcuts PR before merging (the map may have changed).

## 10. Accessibility

`role="tablist"` / `role="tab"` / `aria-selected` / `aria-controls` /
`aria-labelledby`. Arrow keys move among headers; Enter/Space activates;
close affordance keyboard-reachable; dirty state in the accessible name in
words (not `●`/color only); focus moves to the newly active header on close
(or Welcome primary action if none remain); reduced-motion honored.

## 11. i18n (RFC-043)

Add all RFC-057 v2 § 12 keys to English and Japanese in the same PR, sorted, no
untranslated fallback. Plain, non-technical wording. No "Discard" where "Close
without saving" / "Quit without saving" is clearer.

## 12. Testing strategy

### 12.1 omriss-ui unit tests (no Dioxus)

Workspace: empty; open assigns unique `TabId`; active after open/new; close
first/middle/active/last; close/activate unknown `TabId` is `NoSuchTab`;
dirty detection; `dirty_tabs` order; dedupe by identity; untitled never
deduped; stable `TabId` after earlier close; active invariant after every
mutator.

Seam: a **fake** `FocusedDraft` returning `Invalid` proves the switch-block
path, Save-All-block path, and that source text is untouched — **without**
needing JSON/TOML. Markdown editor returns `Applicable`/`Clean`.

### 12.2 app/shell tests

Tab strip hidden at one tab, visible at two; open appends; New appends
untitled; open already-open activates existing; close dirty shows scoped
guard; cancel keeps tab; close-without-saving removes; save-then-close;
`Ctrl+W` closes; `Ctrl+Tab` switches after a valid apply; invalid draft blocks
switch (fake validator).

### 12.3 regression

Single-document open/edit/save unchanged; Markdown byte-preservation golden
tests unchanged; per-tab undo/focus history; per-tab external-modification
check; recent files unchanged; **typing does not re-render tab strip /
inactive tabs** (render-boundary test).

### 12.4 manual QA

Linux/macOS/Windows: shortcuts; tab rendering + overflow; file picker + dedupe;
Save All / quit guard; accessibility focus; long tab names.

## 13. Non-goals

Split panes; multiple windows; tear-off tabs; tab groups; drag-reorder;
session restore; cross-file search; project/workspace files; shared undo;
per-tab search persistence (v1 closes search on switch). Drag-reorder and
session restore are future-RFC candidates.

## 14. Risks and mitigations

- **Workspace signal causes typing re-renders** → keep hot draft local; write
  workspace signal only on structural events (§ 7).
- **Stale modal targets wrong tab** → `TabId` in modal payloads; resolve at
  action time.
- **Save All partial failure** → deterministic order, stop on first failure,
  activate failed tab, preserve saved results.
- **Stale search/status across tabs** → close search and clear status on
  switch.
- **Accidental hard dependency on RFC-050/053** → the seam lives in RFC-057;
  Markdown editor is the only implementor for now.

## 15. Definition of done

- `Workspace`/`Tab`/`TabId`/`DocumentIdentity`/`FocusedDraft` in `omriss-ui`,
  unit-tested (incl. fake-validator seam tests).
- Shell renders the tab strip only with ≥2 tabs; single-document UX unchanged.
- Open/New append; already-open dedupe by identity; untitled never deduped.
- Switch applies valid draft, blocks invalid; Save All validates active only.
- Close/quit guards tab-aware; friendlier labels.
- `preview_open` per-tab; search closes on switch; status active-tab scoped.
- Shortcuts collision-verified; i18n complete in both catalogs.
- Typing does not re-render chrome (render-boundary test passes).
- No `omriss` core change; RFC-001 holds.
- Required gates pass: `cargo fmt --check`, `cargo test`,
  `scripts/check-rfcs.sh`. clippy recommended, not required.
